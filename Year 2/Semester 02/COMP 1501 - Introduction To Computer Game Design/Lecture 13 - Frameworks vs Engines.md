
## Engines

They built the tools to draw pixels to the screen, to efficiently detect when objects collide, to efficiently light up scenes, to handle online multiplayer, to handle input across different computers.

In terms of Godot:  
1. Strong 2D, and weaker but rapidly improving 3D,  
	• It has a different 2D and 3D graphics engine, unlike many other engines  
	• Has more 2D tools built-in than 3D tools.  
2. Intended for small teams with generalized skills,  
	• Has many tools built-in for a variety of disciplines, rather than assuming existing knowledge with other industry tools  
3. Object-Oriented design with composition,  
	• Lots of inheritance and lots of “Can-Do/Has-A” relationships.


**Pros**  
+ Easy to get started with  
+ Able to make complex games  
+ Heavily supported, easy to release to people  

**Cons**  
- Can (should) influence your design  
- Usually have license fees  
- “Generic” implementations may not work for your game

## Frameworks

Game frameworks are programming libraries which provide us with lots of the necessary functions for essential game functionality:  
- Drawing to the screen  
- Playing sound effects  
- Detecting user input  
- Basic user interface elements  
- Basic physics/collision detection


**Pros**  
+ Supports experimental games  
+ Lots of control  
+ Little influence on design  
+ Very powerful for a CS student  

**Cons**  
- Big learning curve  
- Tough to do anything complex  
- Long time to get your game engine built
