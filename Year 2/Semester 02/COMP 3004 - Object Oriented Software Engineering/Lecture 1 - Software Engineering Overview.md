# Software Engineering Aspects

#### **Technical Aspects**
1. **Application Domain**
	- It includes parts of the real world that are relevant to the problem
	- The area of business where the client is an expert
	- Concepts in the application domain often have real-life counterparts
	- The development team must learn about the application domain

2. **Solution Domain**
	- It includes everything related to the solution to the problem
	- Few concepts in the solution domain have real-life counterparts

3. **Models**
	- It’s a representation, or abstraction, of something complex
	- For example, scale models of buildings before they are built


4. **Building Tools**
	- We build models of different aspects of the software system

#### **Management Aspects**
1. Communication Tools
2. Configuration Mangement
3. Rationale Management
4. Software Development Process
5. Impact of Changes
