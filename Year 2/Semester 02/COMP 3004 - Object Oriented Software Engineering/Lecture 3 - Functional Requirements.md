## Non-Functional Requirements

#### 1. Performance
How fast and efficiently the system operates.

**Examples**
- Response time
- Throughput
- Latency
- Resource usage

> _“The system shall respond to user requests within 500 ms.”_


#### 2. Reliability
How consistently the system functions without failure.

**Examples**
- Mean time between failures (MTBF)
- Error rates
- Fault tolerance

> _“The system shall operate with no more than 1 failure per 1,000 hours.”_


#### 3. Availability
When and how often the system is operational.

**Examples**
- Uptime percentage
- Planned downtime windows

> _“The system shall be available 99.9% of the time.”_


#### 4. Security
Protection against unauthorized access or misuse.

**Examples**
- Authentication
- Authorization
- Data encryption
- Auditing

> _“User passwords shall be encrypted at rest and in transit.”_


#### 5. Usability
How easy the system is to learn and use.

**Examples**
- Accessibility
- Learnability
- UI consistency
- Error prevention

> _“New users shall be able to complete registration without training.”_


#### 6. Maintainability
How easy it is to modify, fix, or extend the system.

**Examples**
- Modularity
- Code readability
- Documentation quality

> _“The system shall allow new features to be added without affecting existing modules.”_


#### 7. Scalability
How well the system handles growth.

**Examples**
- Increasing users
- Increased data volume
- Load spikes

> _“The system shall support up to 50,000 concurrent users.”_


#### 8. Potability
How easily the system can run in different environments.

**Examples**
- OS compatibility
- Browser compatibility
- Hardware independence

> _“The application shall run on Windows, macOS, and Linux.”_


#### 9. Compliance/Legal/Regulatory
Adherence to laws, standards, or organizational rules.

**Examples**
- GDPR
- HIPAA
- Accessibility standards
- Industry certifications

> _“The system shall comply with WCAG 2.1 AA standards.”_
